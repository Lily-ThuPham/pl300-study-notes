##### Data hierarchies 
Make sure you know how and where (which views) to create hierarchies, add them to a visual field and apply drill-down. 
To create a hierarchical structure and enable the drill-down function, you can _**use both numerical and categorical data fields**_ in your visuals.
You can create data hierarchy in Power BI by:
- Creating a new hierarchy and adding fields to the hierarchy
- By using DAX (`PATH`)
- By adding fields to the visual axis well 
In Power BI, you can create a maximum of five levels of hierarchy.

###### How DAX objects are stored

| Object                | How it is Stored                                                                                  | When it is Calculated                                                | Impact on Model Size                   |
| --------------------- | ------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- | -------------------------------------- |
| **Measure**           | Measure are not **pre-calculated or stored** in the model.<br>Only the **DAX formula** is stored. | **On the fly, at query time** (when a user interacts with a visual). | **None**.                              |
| **Calculated Column** | Values are materialized and **stored in the model** like any other column.                        | **During data refresh**, once for each row in its table.             | **Increases model size** and uses RAM. |
| **Calculated Table**  | The entire table is materialized and **stored in the model**.                                     | **During data refresh**, after the tables it depends on are updated. | **Increases model size** and uses RAM. |
