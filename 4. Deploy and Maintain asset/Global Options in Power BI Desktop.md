**Global Options** are settings in Power BI Desktop that apply by default to **all** `.pbix` files created or opened on your computer. They control default behaviors for data loading, visuals, and other features, helping to optimize performance and enforce modeling best practices.
**How to Access:** Go to **File > Options and settings > Options**. The "GLOBAL" section is listed in the left pane.

---
### Global Options: Data Load
These settings control default behaviors for importing and refreshing data.

| **Setting Group**              | **Option**                                                                                                                                                                | **Description / Purpose**                                                                                                                                                                                                                                                                    | **Key Implications / Recommendations**                                                                                                                      |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Type Detection**             | _Detect column types and headers for unstructured sources_                                                                                                                | Automatically identifies headers and guesses data types for sources like CSV/Text.                                                                                                                                                                                                           | Saves time but can be inaccurate. **Recommendation:** Keep enabled, but **always verify** types in Power Query.                                             |
| **Relationships**              | _Import relationships from data sources on first load_                                                                                                                    | Automatically creates model relationships based on foreign keys defined in the source database.                                                                                                                                                                                              | Saves time for complex schemas. **Recommendation:** Generally keep enabled.                                                                                 |
|                                | _Update or delete relationships when refreshing data_                                                                                                                     | Checks source for relationship changes during refresh and applies them to the model.                                                                                                                                                                                                         | Keeps model synced but can cause unexpected changes. **Recommendation:** Use with caution; often disabled in mature models.                                 |
|                                | _Autodetect new relationships after data is loaded_                                                                                                                       | Power BI guesses relationships based on column names after data load.                                                                                                                                                                                                                        | Often creates incorrect relationships, leading to errors. **Recommendation:** **Disable** and create relationships manually.                                |
| **Time intelligence**          | _Auto date/time_<br>[Auto date/time in Power BI Desktop - Power BI \| Microsoft Learn](https://learn.microsoft.com/en-us/power-bi/transform-model/desktop-auto-date-time) | Automatically creates hidden date tables/hierarchies for every date column.                                                                                                                                                                                                                  | Convenient for quick analysis but causes model bloat and limitations. **Recommendation:** **Disable** and create a dedicated Date table.                    |
| **Background Data**            | _Allow data previews to download in the background_                                                                                                                       | Fetches data previews asynchronously in Power Query Editor.                                                                                                                                                                                                                                  | Improves PQ editor responsiveness. **Recommendation:** Keep enabled.                                                                                        |
| **Parallel loading of tables** | _Default / One / Custom_                                                                                                                                                  | Controls how many tables Power BI loads simultaneously during refresh.<br>- **Default:** Uses the standard loading method.<br>- **One (disable parallel loading):** Only one table is loaded at a time.<br>- **Custom:** Allows you to set a specific number of tables to load concurrently. | Parallel loading speeds up refresh but can strain resources. **Recommendation:** Start with `Default`; adjust if needed for performance or troubleshooting. |

### Global Options: Report Settings

These settings influence the default appearance and behavior of visuals and interactions in new reports.

| **Setting Group**        | **Option**                                                                     | **Description / Purpose**                                                                         | **Key Implications / Recommendations**                                                                                                                                   |
| ------------------------ | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Visual options**       | _Use the modern visual header..._                                              | Enables the newer header icons that appear inside the visual.                                     | Provides a cleaner look; required for **Grouping**. **Recommendation:** Keep enabled.                                                                                    |
|                          | _Change default visual interaction from cross highlighting to cross filtering_ | Sets the default click action on a visual to filter others instead of highlighting.               | Filtering can be clearer but hides context. **Recommendation:** Usually keep default (highlighting); override per visual pair if needed.                                 |
| **Export data**          | _Allow end users to export..._                                                 | Sets the default permission level (summarized, underlying, none) for exporting data from visuals. | This is only a **default** for new reports. Can be overridden per file and by Admin settings. **Recommendation:** Set based on general policy; always verify per report. |
| **Filtering experience** | _Allow users to change filter types_                                           | Globally enables/disables switching between Basic and Advanced filters in the Filters pane.       | Flexibility vs. potential user confusion. **Recommendation:** Depends on the target audience.                                                                            |
|                          | _Enable search for the filter pane_                                            | Globally enables/disables the search box in the Filters pane.                                     | Improves usability for reports with many filters. **Recommendation:** Keep enabled.                                                                                      |
